Create a detailed outline and structure for a technical blog post.

## Context
- Writer: 5+ years engineering experience
- Readers: 2-3 years experienced engineers — they skim, they push back, they spot hand-waving
- Blog format: Hugo markdown with frontmatter. Each post lives in `content/blogs/<slug>/index.md`

## Your task

The user will provide a topic or title. Do the following:

1. Clarify scope if needed — ask one focused question if the topic is ambiguous
2. Produce a **full outline** with:
   - **Working title** (can be refined later)
   - **One-line hook**: the insight or tension that makes someone read past the intro
   - **TL;DR** (1–2 sentences): what the reader will walk away knowing
   - **Target reader assumption**: what can you take for granted they already know?
   - **Sections** — ordered list with:
     - Section heading
     - 2–4 bullet points on what to cover
     - Any diagrams, code examples, or comparisons that belong here
   - **What to explicitly NOT cover** (scope boundaries keep posts focused)
   - **Suggested categories** (match existing ones from `content/blogs/` where possible)
   - **Multi-part?** If the topic is too deep for one post, propose a series split

3. Flag any sections that will need research or technical verification — those become inputs for `/blog-research`

Present the outline, then ask for feedback before the user moves to writing.
